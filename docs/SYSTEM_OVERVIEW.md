## 2. Architecture Overview

The system utilizes a **Hybrid Architecture** (Strangler Fig Pattern) to migrate from a legacy Next.js backend to a robust Python/FastAPI microservice.

### **Data Flow**
1. **User Request** → `Next.js Frontend` (Port 3000)
2. **Page Rendering** → Server Components (RSC) / Client Components
3. **API Requests** → Proxied via `next.config.js` Rewrites
4. **Backend Processing** → `FastAPI` (Port 8001)
5. **Data Persistence** → `PostgreSQL` & `Redis`

### **Proxy Configuration**
Routes matching `/api/*` are automatically forwarded to the Python backend:
- `/api/auth/*` (Login, Session)
- `/api/courses/*` (Course Management)
- `/api/users/*` (User Management)
